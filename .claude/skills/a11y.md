# Skill: a11y
# Trigger: /skill a11y
# Purpose: audit WCAG 2.1 AA compliance for Meta-CV

## What This Skill Does

When invoked, audit all Astro components for accessibility issues and
propose fixes. Target: WCAG 2.1 Level AA with zero axe-core violations.

## Checklist

### Landmarks & Structure
- [ ] `<header>`, `<main>`, `<footer>`, `<nav>` used correctly (one `<main>` per page)
- [ ] All sections wrapped in `<section>` or `<article>` have an `aria-labelledby`
- [ ] Heading hierarchy is sequential (h1 → h2 → h3, no skips)

### Images & Icons
- [ ] All `<img>` have descriptive `alt` (empty `alt=""` for decorative images)
- [ ] SVG icons used as buttons/links have `aria-label` or `<title>`
- [ ] No `role="img"` without an accessible name

### Keyboard & Focus
- [ ] All interactive elements reachable by Tab
- [ ] Focus ring visible (not `outline: none` without replacement)
- [ ] Skip-to-content link at the top of `<body>`

### Color & Contrast
- [ ] Text contrast ≥ 4.5:1 (normal text) / 3:1 (large text)
- [ ] No information conveyed by color alone

### Print View (`/print`)
- [ ] All content readable without CSS
- [ ] Links show their `href` in print styles

### Forms (if any)
- [ ] Every `<input>` has an associated `<label>`
- [ ] Error messages are programmatically associated

## How to Run a Quick Audit

```bash
# install axe CLI once
pnpm add -D @axe-core/cli

# run against local preview
rtk pnpm build && rtk pnpm preview
npx axe http://localhost:4321 --exit
```

## Reporting Format

```
[A11Y] <WCAG criterion>: <description>
  element: <selector or file:line>
  fix: <one-line description>
```
